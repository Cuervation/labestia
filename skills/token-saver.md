# Token Saver

## Objetivo
Reducir consumo de tokens al trabajar con agentes.

## Reglas
- No reescribir archivos enteros si solo cambia una función.
- Antes de tocar código, listar archivos a modificar.
- Mantener cambios acotados.
- No explicar de más.
- No crear features fuera del pedido.
- No cambiar arquitectura sin motivo.
- No duplicar lógica.
- No pegar documentación enorme en cada respuesta.
- Usar TODOs breves cuando algo queda pendiente.

## Formato
- Archivos modificados
- Qué se cambió
- Cómo probar
- Próximo paso
